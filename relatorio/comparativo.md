# Relatório Comparativo: Abordagem Estruturada vs Orientada a Objetos

## 1. Introdução

Este relatório apresenta as principais diferenças entre a implementação do cadastro de consumidores usando os paradigmas de programação **estruturada** e **orientada a objetos (OOP)** em Java, C# e Python.

---

## 2. Abordagem Estruturada

- Organiza o código por funções/procedimentos e estruturas de dados simples.
- Os dados ficam separados dos comportamentos, manipulados por funções genéricas.
- Exemplo: funções como `cadastrarConsumidor` com listas ou arrays para armazenar dados.

**Vantagens:**  
Simples para programas pequenos; intuitivo em problemas sequenciais.

**Desvantagens:**  
Pouca coesão entre dados e comportamentos; dificulta manutenção e reuso em projetos maiores.

---

## 3. Abordagem Orientada a Objetos (OOP)

- Centraliza dados e comportamentos em classes/objetos.
- Dados e operações são encapsulados em entidades como `Consumidor` e controladores do sistema.
- Métodos pertencem às classes e objetos interagem entre si.

**Vantagens:**  
Maior organização, modularidade e reuso; facilita manutenção e expansão.

**Desvantagens:**  
Mais complexidade inicial para exercícios simples; mais arquivos e abstrações.

---

## 4. Diferenças Práticas Observadas

- **Estruturado:** Funções isoladas, dados em listas. Manipulação simples, mas menos expansível.
- **OOP:** Métodos dentro das classes, entidades claras. Código mais limpo e fácil de evoluir.

---

## 5. Conclusão

Ambas as abordagens funcionam, mas OOP traz ganhos de organização e robustez para sistemas grandes e profissionais, seguindo padrões modernos de desenvolvimento em Java, C# e Python.
